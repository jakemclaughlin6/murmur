  <action>
    The EPUB parser stores raw `src` from `<img>` tags as `ImageBlock.href`. These are often relative to the chapter XHTML file, e.g., `../images/fig1.png` or `../../images/cover.jpg`. The ImageExtractor maps full content paths (`OEBPS/images/fig1.png`), basenames (`fig1.png`), and one-level-stripped paths (`images/fig1.png`) -- but NOT the raw relative paths.

    **Two-sided fix for robustness:**

    **Side 1: ImageExtractor (lib/core/epub/image_extractor.dart)**
    After the existing three mapping lines, add a normalized variant for relative paths. For each `epubHref`, compute `p.normalize(epubHref)` which collapses `../` segments. If the normalized form differs from the already-mapped forms, add it to the mapping. This handles EPUBs where the image path has leading directory traversal.

    Also add: for each segment count from 2 to segments.length-1, map `p.joinAll(segments.sublist(i))` to the output path. This covers EPUBs where images are nested deeply and the `<img>` src references them at varying relative depths. The current code only strips one leading segment.

    **Side 2: _ImageBlockWidget (lib/features/reader/widgets/block_renderer.dart)**
    Add `import 'package:path/path.dart' as p;` at top. In the `build()` method of `_ImageBlockWidget`, after the direct `imagePathMap?[href]` lookup, add a fallback chain:
    1. Try `imagePathMap?[p.normalize(href)]` (collapses `../`)
    2. Try `imagePathMap?[p.basename(href)]` (last resort -- basename only)

    The fallback handles the case where ImageExtractor's map doesn't have the exact relative form but does have the basename. Guard against basename collisions by preferring the direct lookup and normalized lookup first.

    Replace the current logic:
    ```dart
    final localPath = imagePathMap?[href];
    ```
    With:
    ```dart
    final localPath = imagePathMap?[href]
        ?? imagePathMap?[p.normalize(href)]
        ?? imagePathMap?[p.basename(href)];
    ```

    **Tests:**

    In `test/core/epub/image_extractor_test.dart`, add a test verifying that the mapping includes normalized variants. Create a minimal EPUB fixture or mock the extractImages return to verify that `../images/fig.png` resolves.

    In `test/widget/reader/block_renderer_test.dart`, add a test for `_ImageBlockWidget` where the href is `../images/test.png` and the imagePathMap has `images/test.png` mapped -- verify the image widget renders (not the placeholder).
