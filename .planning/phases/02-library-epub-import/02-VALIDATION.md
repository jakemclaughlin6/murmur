## 15-EPUB Test Corpus

The corpus must cover these edge cases to satisfy LIB-03:

| # | Edge Case | Why |
|---|-----------|-----|
| 1 | Standard well-formed EPUB 3 | Baseline |
| 2 | No cover image | LIB-06 fallback icon (D-08) |
| 3 | No author metadata | `books.author` nullable path |
| 4 | Non-ASCII title (e.g., "Les Misérables") | UTF-8 through parser + Drift |
| 5 | Very long (>500 chapters) | Isolate memory / parse duration |
| 6 | Very large file (>50 MB, image-heavy) | File copy cost, cover extraction |
| 7 | Tables in chapter content | D-02 table → paragraph flattening |
| 8 | Footnotes | D-02 footnote → paragraph+marker |
| 9 | Blockquotes + lists | Block IR coverage |
| 10 | Malformed XHTML (unclosed tags) | `package:html` leniency |
| 11 | EPUB 2 (older spec) | epubx backwards compat |
| 12 | EPUB 3 with spine reordering | Non-default chapter ordering |
| 13 | DRM-protected (encryption.xml present) | LIB-04 rejection |
| 14 | Corrupt ZIP (truncated) | LIB-04 exception → snackbar |
| 15 | Multiple cover image candidates | Cover extraction disambiguation |

*All DRM-free entries should come from Project Gutenberg or similar public domain sources.*
