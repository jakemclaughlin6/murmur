```dart
/// Block Intermediate Representation per D-01.
/// Stored as JSON in chapters.blocks_json per D-03.
/// Richer EPUB constructs (tables, footnotes, sidebars) are flattened to
/// nearest equivalents at parse time per D-02.
sealed class Block {
  const Block();
}

class Paragraph extends Block {
  final String text;
  const Paragraph(this.text);
}

class Heading extends Block {
  /// EPUB h1 .. h6 — validated 1..6 inclusive.
  final int level;
  final String text;
  const Heading({required this.level, required this.text});
}

class ImageBlock extends Block {
  /// EPUB-internal href relative to the chapter XHTML.
  final String href;
  final String? alt;
  const ImageBlock({required this.href, this.alt});
}

class Blockquote extends Block {
  final String text;
  const Blockquote(this.text);
}

class ListItem extends Block {
  final String text;
  /// true for <ol>, false for <ul>.
  final bool ordered;
  const ListItem({required this.text, required this.ordered});
}
```

}

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Tampering | block_json.dart fromJson | mitigate | `blockFromJson` throws FormatException on unknown discriminator — prevents silently accepting forged Block types that downstream switches don't handle. |
| T-02-02-02 | Information Disclosure | Block.text fields | accept | Plaintext from EPUB is stored as-is; the EPUB itself is the source of truth for the user, no PII beyond what the user already owns. |
| T-02-02-03 | Denial of Service | blocksFromJsonString (oversized input) | accept | A multi-megabyte chapter JSON is bounded by the parser's chapter size budget (Plan 04). This plan's codec is pure and cannot be attacked without first passing Plan 04's import gate. |
