  <behavior>
    - `TtsQueue(client: TtsClient, cache: TtsCache, player: AudioPlayerHandle)`.
    - Input: `setChapter(bookId, chapterIdx, sentences: List<Sentence>)` — fresh chapter load; eagerly sends `SynthSentence(idx: 0)` for pre-synth (D-11).
    - `play(fromIdx)`: if cache hit for `fromIdx`, `player.setFile` + `play` immediately; send `SynthSentence(idx: fromIdx + 1)` for next pre-synth. If cache miss, send `SynthSentence(idx: fromIdx)` and await `SentenceReady`; upon arrival, setFile + play; send synth for `fromIdx + 1`.
    - On `player.completedStream` event: advance `sentenceIdx` to +1, play next (cache hit expected due to pre-synth), pre-synth +2.
    - `skipForward()`: advance +1; if currently synthesizing N+1, per Pitfall 4 let it complete but discard (no pre-synth kickoff for +2 until the skipped-to sentence is playing). Per RESEARCH: skip latency ≤ remaining synth time.
    - `skipBackward()`: advance -1; if in ring buffer (last 3 played), cache hit → instant play. If not, re-synth.
    - `setVoice(voiceId)`: invalidates cache for current chapter (silently wipes `$cache/{bookId}/{chapterIdx}/`) and re-synths from current sentenceIdx. Send `SetVoice` to client.
    - `setSpeed(s)`: calls `player.setSpeed(s)`. Does NOT send any command to worker (TTS-09 — sherpa stays at 1.0).
    - `pause()`/`resume()`: forwards to player; pre-synth continues in background.
    - `dispose()`: pause player, send Dispose to client, await DisposeAck, close cache handles.
    - Writes sentenceIdx to `playbackStateProvider` as each sentence starts playing (Phase 5 will observe for highlight).
  </behavior>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06-01 | Denial of Service | Pre-synth queue grows unbounded if user rapid-skips | mitigate | Pre-synth is strictly +1 ahead of current; cancel-discards-result gates the queue. |
